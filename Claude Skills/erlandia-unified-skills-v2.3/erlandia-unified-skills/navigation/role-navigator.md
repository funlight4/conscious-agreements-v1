# Erlandia Role Navigator v1.0

## Purpose
Establish clear operational roles that naturally determine whether Opus-level synthesis or Sonnet-level implementation is required. This lightweight navigation tool ensures quality work while optimizing credit usage through conscious role selection.

## Core Principle
```
Different roles require different capabilities.
Architects need synthesis; Builders need consistency.
The role determines the tool, not the task.
Conscious role selection optimizes everything.
When you know your role, you know your model.
```

### Smooth Handoffs:
When switching roles, use transition phrases to maintain context:

**Builder ‚Üí Architect:**
"Elevating from Builder to Architect role. Current implementation reveals need for structural evolution. Recommend Opus."

**Architect ‚Üí Builder:**
"Structure established by Architect. Shifting to Builder role for implementation. Sonnet appropriate."

## Credit Optimization Strategies

### Batching by Role:
- Collect all Architect tasks for single Opus session
- Run Builder tasks continuously in Sonnet
- Schedule Validator checks weekly in Opus

### Role Stacking:
- Start with Architect (Opus) to set structure
- Move to Builder (Sonnet) for extended work
- End with Validator (Opus) to verify
