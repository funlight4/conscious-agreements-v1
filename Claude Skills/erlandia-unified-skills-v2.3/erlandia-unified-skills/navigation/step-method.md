# Step Method v1.0

## Purpose
Master the art of perspective orientation through a fractal visibility system that reveals only what is immediately relevant and achievable at every scale of navigation. This skill transforms overwhelming journeys into clear, manageable progressions by applying the same 3-step visibility rule whether you're planning your next hour or your path to Paradise.

## Core Principle
```
You can only see 3 steps ahead at any scale.
Each step contains substeps following the same rule.
Completion reveals new horizons while maintaining orientation.
The structure is self-similar from substeps to life paths.
Overwhelm dissolves when visibility matches actionability.
Paradise stays at the horizon, pulling forward without demanding direct approach.
```

### Position Definitions

**Step -1: The Anchor**
- What you just completed
- Proof that progress is real
- Reference point preventing "lost" feeling
- Grounds you in accomplishment
- Becomes invisible once Step 0 is well-established

**Step 0: Current Position**
- Where you are RIGHT NOW
- Full visibility of all substeps
- Active engagement zone
- The "you are here" marker
- Complete clarity expected

**Step 1: Next Action**
- Immediately achievable
- Clear enough to act upon
- Within reach from current position
- Visible substeps (following visibility gradient)
- The focus of your attention

**Step 2: Coming Soon**
- Visible but not yet actionable
- Provides context for Step 1
- Partial substep visibility
- Creates anticipation
- Helps sequence priorities

**Step 3: Horizon**
- Edge of visibility
- Paradise beckons from here
- Minimal substep visibility (structure only)
- Aspirational pull
- Aligns with PGA principle: goal visible but not targeted directly

**Step 4+: Beyond**
- Hidden by design
- Trust the process
- Will be revealed when appropriate
- No cognitive load required
- Faith in the unfolding

## Working ON Mode (Design)

### Designing Step-Based Systems

When creating any progressive system, apply these principles:

**Step 1: Define the Macro Steps**
```
What are the 4-7 major milestones from start to destination?
These become your Steps 0 through 3+ at the highest level.
```

**Step 2: Create Substep Structure**
```
For each major step, what are the 3-5 substeps?
Apply the same visibility gradient within each.
```

**Step 3: Design Revelation Points**
```
What gets revealed when each step completes?
- New substep visibility in current step
- New step visibility at horizon
- Achievement/celebration moment
```

**Step 4: Build Navigation Interface**
```
How will users see their Step Method view?
- Current position clearly marked
- Visible range appropriately limited
- Progress indicators for each step
- Anchor (Step -1) visible for grounding
```

**Step 5: Test Cognitive Load**
```
At any given moment, can user answer:
- Where am I? (Step 0)
- What's next? (Step 1)
- Where did I come from? (Step -1)
- Where am I going? (Step 3)

If any answer is unclear or overwhelming, adjust visibility.
```

## Common Mistakes and Solutions

### Mistake 1: Showing Too Much
**Pattern:** Revealing Step 4, 5, 6 or all substeps at once
**Problem:** Overwhelm, anxiety, paralysis
**Solution:** Strictly enforce 3-step visibility; hide what's beyond

### Mistake 2: No Anchor Visibility
**Pattern:** Not showing Step -1 (what was completed)
**Problem:** Feeling lost, ungrounded, "where did I come from?"
**Solution:** Always show recent completion as anchor

### Mistake 3: Uniform Substep Visibility
**Pattern:** Showing all substeps for all visible steps equally
**Problem:** Still too much information; breaks gradient
**Solution:** Apply visibility gradient - fewer substeps visible as distance increases

### Mistake 4: No Horizon
**Pattern:** Only showing immediate next step, no glimpse of future
**Problem:** No aspirational pull, unclear "why" of current action
**Solution:** Always have Step 3 horizon visible, even if just structure

### Mistake 5: Breaking Fractal Coherence
**Pattern:** Different rules at different levels
**Problem:** Confusion, inconsistency, loss of trust in system
**Solution:** Same 3-step rule at EVERY level of granularity

### Mistake 6: No Revelation Celebration
**Pattern:** Step completion just leads to more steps, no acknowledgment
**Problem:** Endless treadmill feeling, no sense of progress
**Solution:** Mark completions with celebration, achievement, XP
