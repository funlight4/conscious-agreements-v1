# Working IN vs ON Navigation v1.0

## Purpose
Provide conscious navigation at every decision intersection between continuing to work IN the current system versus stepping back to work ON the system itself. This meta-awareness prevents both stagnation (only working IN) and paralysis (only working ON).

## Core Principle
```
Every tension is a portal offering two paths:
Working IN = Fulfill current purpose with current structure
Working ON = Evolve structure to better serve purpose
Wisdom knows which path serves evolution NOW.
```

## The Two Modes Defined

### Working IN Mode
**Purpose:** Fulfill existing container's purpose with current structure

**Characteristics:**
- Using current systems/processes
- Following established patterns
- Maintaining flow within boundaries
- Producing intended outputs
- Operating at current capacity

**When Optimal:**
- Structure serves purpose well
- Learning still being extracted
- Step distance remains ≤3
- PP sufficient for beauty
- Energy for execution available

### Working ON Mode
**Purpose:** Evolve container structure to better serve purpose

**Characteristics:**
- Questioning current systems
- Redesigning patterns
- Adjusting boundaries
- Upgrading capacity
- Creating new containers

**When Optimal:**
- Tensions repeatedly arise
- Same problems recur
- Purpose outgrows structure
- PP could significantly increase
- Learning integration complete

## The Navigation Framework

### Step 1: Tension Recognition
Every tension/problem is a decision portal:
```
TENSION ARISES
    ↓
PAUSE - Meta-awareness activated
    ↓
Two paths revealed
```

### Step 2: Mode Assessment
Quick diagnostic:

| Indicator | Work IN | Work ON |
|-----------|---------|---------|
| Can current structure handle this? | Yes → IN | No → ON |
| Is this a recurring pattern? | No → IN | Yes → ON |
| Will solving this way create beauty? | Yes → IN | No → ON |
| Are we within 3 steps of purpose? | Yes → IN | No → ON |
| Is learning still being extracted? | Yes → IN | No → ON |

### Step 3: Step Method Integration

#### Working IN Navigation:
```
Current Position (Step 0)
    ↓
Solution within system (Step 1)
    ↓
Implementation (Step 2)
    ↓
Completion (Step 3 MAX)
```

#### Working ON Navigation:
```
Current Position (Step 0)
    ↓
Step back to view system (Step 1)
    ↓
Redesign structure (Step 2)
    ↓
Implement new system (Step 3 MAX)
```

### Step 4: Decision Validation

Before committing to path:

**IN Mode Validation:**
- Will this actually solve it?
- Or just postpone the tension?
- Is energy well spent here?
- Does this increase PP?

**ON Mode Validation:**
- Is this the right time?
- Do we have resources?
- Will evolution serve purpose?
- Are we avoiding real work?
