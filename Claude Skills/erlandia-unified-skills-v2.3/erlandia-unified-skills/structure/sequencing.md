## Universal Sequencing Principles

### Principle 1: Dependencies First
```
Before X can happen, Y must exist
Example: Before building house, need foundation
Application: Map all dependencies before sequencing
```

### Principle 2: Energy Optimization
```
Sequence by energy availability
Morning: High cognitive load
Afternoon: Collaborative work
Evening: Integration/reflection
```

### Principle 3: Truth Level Alignment
```
Sequence from higher truth to lower implementation
L7-L5: Vision/Purpose (First)
L4-L3: Systems/Principles (Second)
L2-L1: Actions/Tasks (Third)
```

### Principle 4: Container Before Contents
```
Create structure before filling
Example: Create filing system before filing
Application: Design container, then populate
```

### Principle 5: Simple Before Complex
```
Master basics before advanced
Example: Crawl → Walk → Run → Dance
Application: Build foundation systematically
```

## Common Sequencing Errors

### Error 1: Skipping Installation
**Pattern:** Jump straight to use without learning
**Result:** Incompetent action, frustration
**Fix:** Honor installation phase fully

### Error 2: Perpetual Installation
**Pattern:** Always learning, never applying
**Result:** Knowledge without wisdom
**Fix:** Set use phase triggers

### Error 3: Wrong Level Sequence
**Pattern:** Start with details, miss vision
**Result:** Efficient dysfunction
**Fix:** Sequence from high to low truth

### Error 4: Ignoring Dependencies
**Pattern:** Try step 5 before step 2
**Result:** Constant backtracking
**Fix:** Map dependencies first

### Error 5: Energy Misalignment
**Pattern:** Creative work when exhausted
**Result:** Poor quality, burnout
**Fix:** Match task to energy
