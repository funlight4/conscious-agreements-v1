## Program Tier Specification *(NEW)*

### Why Tier Specification Matters

Every program should clearly specify its minimum Installation Requirements tier. This:
- Sets appropriate expectations
- Ensures users have necessary "hardware"
- Prevents frustration from mismatched experiences
- Enables proper journey mapping

### Tier Labeling System

All programs should display their tier requirement:

```
┌─────────────────────────────────────────┐
│  PROGRAM NAME                           │
│  ─────────────────────────────────────  │
│  Minimum Tier: [MINIMUM/RECOMMENDED/    │
│                 OPTIMAL]                │
│  ─────────────────────────────────────  │
│  Description...                         │
└─────────────────────────────────────────┘
```

### Tier Requirement Guidelines

**MINIMUM TIER Programs:**
- F1: Healthy Boundaries (entry)
- Introduction to Erlandia
- Basic concept programs
- Entry-level Paradise Path content
- Foundation building experiences

Characteristics:
- No prerequisites beyond minimum requirements
- Simple, accessible language
- Maximum support and scaffolding
- Celebrates beginner progress

**RECOMMENDED TIER Programs:**
- F2-F9 Fundamentals
- Full Paradise Path curricula
- Integration programs
- Group masterclasses
- Intermediate to advanced content

Characteristics:
- Assumes growth orientation established
- Can use full Erlandia vocabulary
- Normal pacing and expectations
- Deep integration focus

**OPTIMAL TIER Programs:**
- Mastery programs
- Certification tracks
- Teaching preparation
- Leadership development
- Paradise Engineering
- 1-on-1 intensive mentorship

Characteristics:
- Requires extensive foundation
- Rapid pace options
- Maximum depth
- Service/teaching orientation
- Full system synthesis

### Program Tier Assessment Checklist

Before finalizing program tier, verify:

**For Minimum Tier:**
- [ ] Accessible to someone just starting growth journey
- [ ] No assumed vocabulary or concepts
- [ ] Maximum support built in
- [ ] Clear, simple success markers
- [ ] Path forward to more advanced programs

**For Recommended Tier:**
- [ ] Assumes basic growth orientation
- [ ] Can reference other Fundamentals
- [ ] Integration depth appropriate
- [ ] Connects to Paradise Path journey
- [ ] Prepares for leadership eventually

**For Optimal Tier:**
- [ ] Requires demonstrated mastery foundation
- [ ] Teaching/service component
- [ ] Maximum depth and synthesis
- [ ] Certification or leadership track
- [ ] Paradise Engineering readiness

## Program Types & Templates

### Fundamental Programs
**Purpose:** Foundation building
**Structure:** Linear progression
**Duration:** 4-12 weeks
**Delivery:** Self-paced with support
**Typical Tier:** Minimum (F1), Recommended (F2-F9)

**Template:**
```
Week 1-2: Exposure & Context
Week 3-4: Exploration & Understanding
Week 5-8: Practice & Application
Week 9-12: Integration & Embodiment
```

### Journey Programs
**Purpose:** Complete transformation path
**Structure:** Modular with branches
**Duration:** 3-12 months
**Delivery:** Guided cohort
**Typical Tier:** Recommended

**Template:**
```
Module 1: Awakening (Entry point)
Module 2: Foundation (Fundamentals)
Module 3-6: Journey (Path specific)
Module 7: Integration (Synthesis)
Module 8: Service (Giving forward)
```

### Intensive Programs
**Purpose:** Rapid transformation
**Structure:** Immersive experience
**Duration:** 3-30 days
**Delivery:** Live intensive
**Typical Tier:** Recommended to Optimal

**Template:**
```
Phase 1: Breaking patterns
Phase 2: Installing new
Phase 3: Integration
Phase 4: Activation
```

### Mastery Programs
**Purpose:** Deep expertise
**Structure:** Spiral deepening
**Duration:** 6-24 months
**Delivery:** Mentorship + practice
**Typical Tier:** Optimal

**Template:**
```
Cycle 1: Foundational mastery
Cycle 2: Advanced application
Cycle 3: Teaching preparation
Cycle 4: Innovation contribution
```

## Common Development Pitfalls

### Pitfall 1: Information Overload
**Problem:** Too much content
**Solution:** Focus on integration depth
**Tool:** 70/30 Known/Unknown ratio

### Pitfall 2: Linear Rigidity
**Problem:** No adaptation space
**Solution:** Build in flexibility
**Tool:** Multiple journey paths

### Pitfall 3: Missing Mystery
**Problem:** All Known, no Unknown
**Solution:** Add emergence space
**Tool:** Unknown integration

### Pitfall 4: Low Purpose Power
**Problem:** Single purpose only
**Solution:** Find additional purposes
**Tool:** HDM calculation

### Pitfall 5: Transformation Bypass
**Problem:** Information without transformation
**Solution:** Add practice and integration
**Tool:** Sequencing framework

### Pitfall 6: Tier Mismatch *(NEW)*
**Problem:** Content doesn't match specified tier
**Solution:** Realign content to tier or adjust tier
**Tool:** Tier Assessment Checklist
